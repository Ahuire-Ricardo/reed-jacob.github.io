Local Storage

June 3rd, 2015

Local storage enables web applications to store data locally within the user's browser. The browser has an allotted amount of space that is dedicated to storing user data. This information includes, but is not limited to: user preference settings, application data and other more temporary forms of relevant user data. HTML5 local storage provides two objects for storing data in the client:

window.sessionStorage (Stores data for a single session)
window.localStorage (Stores data with no expiration date)

Let's utilize the window.sessionStorage object in order to store some information temporarily! Here is our first provided example from the web:

if( sessionStorage.hits ){
  sessionStorage.hits = Number(sessionStorage.hits) +1;
} else {
  sessionStorage.hits = 1;
}
document.write("Total Hits: " + sessionStorage.hits );

Output:

Refresh the page to increase number of hits.
Close the window / tab and open it again and check the result.

Now we will try the same code but with the window.localStorage object in order to store information indefinitely:

if( localStorage.hits ){
  localStorage.hits = Number(localStorage.hits) +1;
} else {
  localStorage.hits = 1;
}
document.write("Total Hits: " + localStorage.hits );

Output:

Refresh the page to increase number of hits.
Close the window / tab and open it again and check the result.

Let's not forget the all important clear button. :^) If you'd like to implement a way to clear the user's local storage, you can simply use this object:

window.localStorage.clear()
