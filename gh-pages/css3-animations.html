CSS3 Animations

May 28th, 2015

Animations vs. Transitions

CSS3 Animations share many similarities with our previous topic, transitions. For example, both specify properties to be changed, specify a duration to control how long a animation / transition will take, and visually change webpage properties in real time. The main difference between the two is the method in which these actions are triggered and the ability to loop. Transitions require some form of input from the user, be it pressing a button or hovering over something. Animations on the other hand can be set to start automatically and continue to play or "loop".

Transitions are best suited for simple from / to animations. Animations are best suited for more complex animations that include multiple keyframes (which are essentially points in which the animation changes). To get a better look at what properties can be animated, check here: w3.org. To see some of these animations in action, let's take a look at some examples:

Here's the first example!

.slidein {
      -moz-animation-duration: 3s;
      -webkit-animation-duration: 3s;
      -moz-animation-name: slidein;
      -webkit-animation-name: slidein;
      -moz-animation-iteration-count: 5;
      -webkit-animation-iteration-count: 5;
      -moz-animation-direction: alternate;
      -webkit-animation-direction: alternate;
    }
    
    @-moz-keyframes slidein {
      from {
        margin-left:100%;
        width:300%
      }
      
      to {
        margin-left:0%;
        width:100%;
      }
    }
    
    @-webkit-keyframes slidein {
      from {
        margin-left:100%;
        width:300%
      }
      
      to {
        margin-left:0%;
        width:100%;
      }
    }
